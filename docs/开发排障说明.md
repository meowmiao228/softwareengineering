# 本地开发与故障排查说明

本文档汇总在本项目中常见的配置与排障操作（含 IntelliJ IDEA、端口占用、大模型 API 密钥等）。

---

## 1. 大模型 API 密钥（默认 DeepSeek）

项目通过 `application.yml` 中的 `llm` 段接入 **OpenAI 兼容** 的 Chat Completions 接口，默认指向 **DeepSeek**（见 [DeepSeek API 文档](https://api-docs.deepseek.com/)）。**不要将真实密钥写入仓库或 YAML 文件**：

```yaml
llm:
  api-key: ${DEEPSEEK_API_KEY:${LLM_API_KEY:}}
```

优先读取 **`DEEPSEEK_API_KEY`**；未设置时再读 **`LLM_API_KEY`**（便于与其他工具统一变量名）。

### 1.1 IntelliJ IDEA

1. **运行 → 编辑配置…**，选中 Spring Boot 配置（如 `AicaoApplication`）。
2. 点击 **「修改选项」**，勾选 **「环境变量」**。
3. 在环境变量中增加：`DEEPSEEK_API_KEY=你的密钥`（或 `LLM_API_KEY=你的密钥`）。
4. **应用 → 确定** 后，**停止并重新运行**应用，配置才会生效。

### 1.2 PowerShell（当前终端会话）

```powershell
$env:DEEPSEEK_API_KEY = "你的密钥"
.\mvnw.cmd spring-boot:run
```

### 1.3 Windows 系统环境变量（长期生效）

「编辑系统环境变量」→「环境变量」→ 新建用户或系统变量：名称 **`DEEPSEEK_API_KEY`**，值为密钥 → 确定后**重新打开**终端或 IDEA。

### 1.4 可选：修改服务商地址与模型（含改回 Kimi）

编辑 `src/main/resources/application.yml` 中的 `llm` 段，例如：

- **DeepSeek（默认）**：`api-base-url: https://api.deepseek.com`，`chat-completions-path: /chat/completions`，`model` 如 `deepseek-v4-flash` 或 `deepseek-v4-pro`。
- **Kimi（月之暗面）**：`api-base-url: https://api.moonshot.ai/v1`，`chat-completions-path: /chat/completions`，`model` 填 Kimi 控制台中的模型名；密钥在 IDEA 中配置为对应平台的环境变量，并在 YAML 里把 `api-key` 改为 `${你的变量名:}`，或继续用 `LLM_API_KEY` 在运行配置里注入。

修改后**重启**应用。

## 2. 端口 8080 已被占用（Web server failed to start）

现象：启动日志中出现 `Port 8080 was already in use`，进程退出码为 `1`。

常见原因：本机已有另一个 Spring Boot、Tomcat、或此前用命令行启动的同一项目仍占用 8080。

### 2.1 查找占用 8080 的进程（PowerShell）

```powershell
Get-NetTCPConnection -LocalPort 8080 -ErrorAction SilentlyContinue |
  Where-Object State -eq Listen |
  Select-Object LocalAddress, LocalPort, State, OwningProcess
```

记下 **`OwningProcess`**（即 PID）。

### 2.2 结束该进程

```powershell
Stop-Process -Id <PID> -Force
```

将 `<PID>` 替换为上一步得到的数字。

### 2.3 再次确认端口已释放

```powershell
Get-NetTCPConnection -LocalPort 8080 -ErrorAction SilentlyContinue
```

无输出表示当前无监听，可重新在 IDEA 或命令行启动应用。

### 2.4 改用其他端口（需同时改前端代理）

若希望后端监听例如 `8081`：

1. 在 `application.yml` 中设置：

   ```yaml
   server:
     port: 8081
   ```

2. 修改 `frontend/vite.config.ts` 中 `server.proxy['/api'].target` 为 `http://localhost:8081`。

3. 重新启动后端与前端开发服务器。

```powershell
cd c:\Users\xuyidong\IdeaProjects\aicao
```

若命令行可启动而 IDEA 失败，重点检查 IDEA 的 **JDK 版本（建议 17）**、**Maven 重新导入**、**运行配置中的主类**（`org.example.aicao.AicaoApplication`）以及 **环境变量** 是否勾选并填写正确。

---

## 4. 关于「后台 spring-boot:run 退出码为 1」

若通过 IDE 或工具**强制结束**正在由 `mvnw spring-boot:run` 拉起的 Java 进程，Maven 可能报告 `Process terminated with exit code: -1` 或构建失败。这通常表示**子进程被终止**，不一定是业务代码错误；释放端口后直接在 IDEA 中正常运行即可。

---

## 5. 快速自测接口（应用已启动后）

在 PowerShell 中（注意 JSON 使用双引号）：

```powershell
Invoke-RestMethod -Uri "http://localhost:8080/api/v1/teaching/sessions" `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"title":"测试","subjectCode":"MATH"}'
```

若返回包含 `id`、`title` 等字段的 JSON，说明后端与数据库初始化正常。调用对话接口还需已正确配置 **`DEEPSEEK_API_KEY`**（或 `LLM_API_KEY`）。

## 6. 前端开发服务

```powershell
cd frontend
npm install
npm run dev
```

浏览器访问 `http://localhost:5173`；开发模式下 `/api` 会代理到后端（默认 `http://localhost:8080`）。
